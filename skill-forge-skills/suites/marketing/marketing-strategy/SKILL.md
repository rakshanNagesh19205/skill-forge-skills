## Section 1: The Positioning Foundation

Before any marketing activity, you need to answer these questions:

### The Positioning Statement
```
FOR [target customer]
WHO [has this problem/need]
[Product name] IS A [category]
THAT [key benefit/differentiation]
UNLIKE [primary alternative]
OUR PRODUCT [primary differentiation]
```

**Example:**
```
FOR solo developers
WHO need code reviews but don't have teammates
CodeReview Pro IS A code analysis tool
THAT catches bugs and suggests improvements automatically
UNLIKE traditional code review tools that require human reviewers
OUR PRODUCT gives you expert-level feedback in seconds, 24/7
```

### The One-Sentence Pitch
```
FORMULA: I help [specific audience] [achieve outcome] without [pain point]

EXAMPLES:
- "I help freelancers get paid faster without chasing invoices"
- "I help solo devs ship better code without waiting for reviews"
- "I help remote teams stay aligned without more meetings"
```

## Section 9: Guardrails

### What This Skill Does NOT Do
- Promise specific results or timelines
- Recommend paid acquisition for bootstrapped solo devs
- Create unrealistic marketing plans
- Suggest channels that don't fit solo constraints
- Ignore product-market fit issues

### Quality Floors
- Every strategy recommendation must be executable by one person
- Time estimates must be realistic (include buffer)
- Channel recommendations must match stated ICP
- Plans must prioritize ruthlessly (no "do everything" advice)

### Boundaries
- Will note when product needs work before marketing
- Will recommend fewer channels, not more
- Will suggest simplifying when plans are too complex
- Will acknowledge when paid help might be necessary
