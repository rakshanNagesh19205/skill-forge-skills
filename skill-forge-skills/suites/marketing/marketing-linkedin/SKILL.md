## Section 2: Hook Formulas That Work

The first 2-3 lines determine whether anyone reads the rest. These formulas consistently perform:

### Pattern 1: Contrarian/Controversial Take
```
[Common belief] is wrong.

Here's what actually works:

[Your insight]
```
**Example:**
```
"Move fast and break things" is terrible advice for solo devs.

Here's why slow and steady wins:

I spent 3 weeks on something I could've shipped in 3 days...
```

### Pattern 2: Confession/Vulnerability
```
I [embarrassing admission].

[What I learned from it]
```
**Example:**
```
I mass-emailed 500 people with "[First Name]" in the subject line.

Turns out my mail merge was broken.

Here's what the 47 people who replied taught me about recovering from mistakes...
```

### Pattern 3: Specific Number + Outcome
```
[Specific number] [timeframe] of [activity].

Here's what [specific result]:

[Breakdown]
```
**Example:**
```
200 days of building in public.

Here's what actually moved the needle:

- Days 1-50: [insight]
- Days 51-100: [insight]
- Days 101-200: [insight]
```

### Pattern 4: Behind-the-Scenes
```
Everyone sees [visible success].

Nobody sees [hidden reality]:

[Honest breakdown]
```
**Example:**
```
Everyone sees: "Hit $5K MRR!"

Nobody sees:
- 14 months of $0
- 3 failed product pivots
- 1 near-complete burnout

Here's the timeline nobody posts about...
```

### Pattern 5: Unexpected Comparison
```
[Your professional topic] is like [unexpected analogy].

Here's why:

[Insight that makes the comparison land]
```
**Example:**
```
Debugging code is like solving a murder mystery.

You have:
- A victim (broken feature)
- Suspects (recent commits)
- Evidence (error logs)
- Red herrings (unrelated warnings)

Here's my detective framework for finding bugs faster...
```

### Pattern 6: Direct Address + Challenge
```
If you're [specific situation], stop [common mistake].

Instead:

[Better approach]
```
**Example:**
```
If you're building a SaaS, stop adding features.

Instead, talk to your churned users.

I just did 15 churn interviews. Here's what they revealed...
```

## Section 3: Post Structures

### Structure A: The Story Post (Best for engagement)
```
[HOOK: 1-2 lines that create curiosity]

[CONTEXT: Quick setup, 2-3 sentences]

[TURNING POINT: What changed/went wrong/surprised you]

[LESSON: The insight or takeaway]

[CTA: Question or invitation to discuss]
```

**Length:** 200-300 words
**Best for:** Personal experiences, failures, wins, learnings

### Structure B: The List Post (Best for saves/shares)
```
[HOOK: Promise of value]

[LIST: 5-10 items, each 1-2 sentences]

[BONUS: One extra insight]

[CTA: Ask which resonates most]
```

**Length:** 150-250 words
**Best for:** Tips, tools, frameworks, processes

### Structure C: The Hot Take (Best for comments)
```
[CONTRARIAN HOOK: Challenge conventional wisdom]

[EVIDENCE: Why you believe this, 2-3 points]

[NUANCE: Acknowledge the other side]

[INVITATION: Ask for other perspectives]
```

**Length:** 100-200 words
**Best for:** Industry opinions, trend commentary, philosophy

### Structure D: The Build-in-Public Update
```
[MILESTONE/UPDATE: What happened]

[NUMBERS: Specific metrics]

[LEARNING: What this taught you]

[NEXT: What you're working on]

[ASK: Specific question for feedback]
```

**Length:** 150-200 words
**Best for:** Regular updates, transparency posts

## Section 10: Guardrails

### What This Skill Does NOT Do
- Create inauthentic engagement bait
- Write corporate jargon or buzzword soup
- Produce "LinkedIn Lunatic" parody content
- Make unrealistic promises about virality
- Encourage posting frequency over quality

### Quality Floors
- Every post must have a genuine insight or value
- Hooks must be honest (not clickbait that doesn't deliver)
- CTAs must be relevant and non-spammy
- Personal stories must have a transferable lesson

### Boundaries
- Will not write posts that misrepresent results
- Will not create fake "I got fired and here's what I learned" stories
- Will not produce content designed purely to game the algorithm
- Will suggest when a topic isn't right for LinkedIn
